# AIMetodolog

## Описание проекта

AIMetodolog — это инновационная система для создания и управления учебными занятиями с использованием искусственного интеллекта. Проект предназначен для преподавателей, методистов и образовательных организаций, позволяя автоматизировать процесс разработки учебных материалов с сохранением педагогического качества и гибкости.

## Ключевые возможности

### 1. Режим создания занятия в диалоге
Система позволяет создавать учебные занятия через интерактивный диалог с языковой моделью (LLM). Пользователь может описывать цели, темы и требования, а система генерирует структурированный план занятия с учетом педагогических принципов.

### 2. Переключение режима верстки по разным уровням структуры занятия
Пользователь может переключать детализацию генерации между различными уровнями структуры занятия ('full': 'Генерация занятия целиком', 'sections': 'По разделам 1 уровня', 'subsections': 'По подразделам') для оптимального представления материала в зависимости от целей, аудитории, объема, степени проработки или в целях экономии токенов.

### 3. Корректировка сгенерированной LLM структуры занятия
Предусмотрены инструменты для ручной корректировки сгенерированной структуры занятия. Пользователь может добавлять, удалять или изменять элементы, а также переупорядочивать блоки для достижения наилучшего результата.

### 4. Подключение разных моделей
Система поддерживает интеграцию с различными языковыми моделями (например, OpenAI GPT, Anthropic Claude, локальные модели и т.д.), что позволяет выбирать наиболее подходящую модель для конкретной задачи. В настоящий момент реализован выбор моделей между моделями:
'gemini': 'google/gemini-2.5-flash-lite',
'deepseek-free': 'tngtech/deepseek-r1t2-chimera:free',
'llama': 'meta-llama/llama-3.3-70b-instruct:free'

### 5. Локальный "скриптовый" режим и режим notebook Google Colab
- **Локальный скриптовый режим**: позволяет запускать систему на локальной машине с использованием Python-скриптов для максимальной производительности и контроля.
- **Режим Google Colab**: обеспечивает удобную работу в облачной среде с возможностью использования GPU и совместной работы(прямо сейчас в Google Colab не тестил, требует коррекции, но можно сделать легко, главное правильно импортировать инфраструктуру и запустить внутри mian.py :) Местный файл main.ipynb лучше даже не трогать).

### 6. Режимы дебагинга и логирования
- **Расширенное логирование**: запись всех этапов генерации, запросов к LLM и результатов для последующего анализа.
- **Интерактивный дебагинг**: возможность пошагового выполнения и проверки промежуточных результатов.
- **Визуализация процессов**: отображение структуры занятия и связей между элементами для лучшего понимания работы системы.
Гибкая настройка деморежимов:
# Демо-режимы для экономии ресурсов при тестировании
# Устанавливаются вручную разработчиком перед запуском

# Локальное тестирование без подключения к LLM
DEMO_LOCAL = False  # TRUE/FALSE (вместо ответа LLM выводится заранее приготовленный шаблон)

# Локальное тежим для локальной русифицированной контурной модели LLM
DEMO_LOCAL_LLM = False  # TRUE/FALSE (запрос к локальной русифицированной контурной модели LLM)

# Локальное тестирование с подключением к большой модели LLM, но с запросом-заглушкой
DEMO_BIG_LLM = False  # TRUE/FALSE (подставляется тестовый запрос для экономии трафика)

# Локальное тестирование с реальным подключением к большой модели LLM и с логированием
DEMO_BIG_LLM_REAL = True  # TRUE/FALSE (реальный запрос с логированием сырых данных)

## План расширения системы

### 1. Выбор разных специалистов и диалогов
Будут добавлены предустановленные профили специалистов (методист, предметный эксперт, дизайнер учебных материалов и др.) с соответствующими диалоговыми стратегиями для более точной генерации.
# Выбор разных режимов генерации контента

### 2. Выбор разного инструментария при создании занятия
Интеграция дополнительных инструментов: генерация упражнений, создание презентаций, подбор мультимедийных материалов, формирование оценочных средств.

### 3. Подключение локальных и удаленных баз данных по технологии RAG
- Использование Retrieval-Augmented Generation (RAG) для доступа к базам знаний, учебным материалам и методическим ресурсам.
- Поддержка как локальных хранилищ, так и облачных баз данных для расширения контекста генерации.

### 4. Гибкое переключение между контурной, бесплатной и платными моделями LLM
- **Контурная модель**: быстрая и экономичная генерация базовой структуры.
- **Бесплатные модели**: использование открытых моделей для снижения затрат.
- **Платные модели**: доступ к наиболее продвинутым моделям для сложных задач.

### 5. Использование разных моделей для разных задач/разделов
Возможность назначения специализированных моделей для конкретных этапов создания занятия (например, одна модель для генерации структуры, вторая — для генерации заданий, третья - гля генерации графиков в задачах по физике и т.п.).

### 6. Подписание занятий несмываемыми авторскими правами
Внедрение механизмов цифровой подписи и блокчейн-технологий для защиты интеллектуальной собственности и подтверждения авторства созданных материалов.

### 7. Дополнительные возможности
- Экспорт занятий в различные форматы (PDF, DOCX, HTML, SCORM).
- Интеграция с системами дистанционного обучения (LMS).
- Коллаборативная работа над занятиями.
- Аналитика использования и эффективности сгенерированных материалов.

## Технологический стек
- Python 3.8+
- LlamaIndex для взаимодействия по технологии RAG с разными LLM
- LangChain / LlamaIndex для оркестрации работы с LLM в диалогах
- FastAPI / Streamlit для веб-интерфейса
- SQLite / PostgreSQL для хранения данных
- Docker для контейнеризации

## Установка и запуск

python3 main.py - запускает корень диалога создания нового занятия. 

Нажатие в диалоге на Enter вводит значение по умолчанию, кроме имени будущего файла .ipynb. Это имя надо ввести вручную. Результат сохранится в папке output.
(Инструкции по установке и запуску будут добавлены в следующей версии)
